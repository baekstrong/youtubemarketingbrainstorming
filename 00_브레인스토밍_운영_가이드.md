# 유튜브 마케팅 팀 브레인스토밍 운영 가이드

## 채널 개요
- **채널명**: 백관장 (유튜브 5만+ 구독자)
- **사업명**: 근력학교 (고대 앞, 안암)
- **채널 주제**: 직장인 체력 향상
- **타겟 시청자**: 30~40대 사무직 직장인, 운동 경험 거의 없고 체력 저하를 체감하는 사람
- **채널 톤**: 옆자리 동료가 알려주는 느낌, 현실 밀착, 작은 실천 유도
- **USP**: "저질 체력 → 강철 체력, 주 2회 3시간으로"

## 브레인스토밍 시작 전: 모드 선택 (필수)

**주제가 입력되면 브레인스토밍을 시작하기 전에 반드시 아래 선택지를 사용자에게 제시한다.**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  브레인스토밍 모드를 선택해주세요
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [A] 레퍼런스 참고 모드
      → 기존 콘텐츠·톤·전략·글쓰기 프레임워크를 반영
      → 채널 일관성 유지, 기존 콘텐츠와 시리즈 연결에 유리
      → 중복 콘텐츠 자동 방지

  [B] 프레쉬 모드
      → 레퍼런스 완전 무시, 백지에서 출발
      → 기존 패턴에서 벗어난 새로운 아이디어 발굴에 유리
      → 단, 기존 콘텐츠와 중복될 수 있음

  [C] 둘 다 보기
      → A, B 두 가지 결과를 모두 생성
      → 비교 후 더 나은 방향 선택 가능
      → 시간이 2배 걸림

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  A, B, C 중 하나를 선택해주세요:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 모드별 동작 방식

**[A] 레퍼런스 참고 모드**
- 모든 에이전트가 `reference/` 폴더 내 파일을 참조
- 기존 콘텐츠 목록을 확인하여 중복 주제 자동 회피
- 글쓰기 프레임워크(제목 짓기 6가지, SEDA, 대본 구조)를 적용
- 마케팅 퍼널과 연결되는 수익화 전략 제안
- 채널 톤("백관장이었어요" 마무리 등) 유지

**[B] 프레쉬 모드**
- `reference/` 폴더를 참조하지 않음
- 에이전트 각자의 일반적 전문성만으로 아이디어 발산
- 기존 패턴·공식에 얽매이지 않는 자유로운 발상
- 새로운 포맷, 새로운 타겟 확장, 실험적 콘텐츠에 적합

**[C] 둘 다 보기**
- A 모드 결과를 먼저 생성
- B 모드 결과를 이어서 생성
- 마지막에 두 결과의 차이점과 각각의 강점을 비교 요약
- 사용자가 원하는 방향을 선택하거나 조합 가능

## 브레인스토밍 프로세스

### Phase 1: 주제 투입 & 리서치 (크리에이티브 디렉터 + 리서처)
```
1. 크리에이티브 디렉터가 주제를 프레이밍한다
   → "이 주제를 왜 다뤄야 하는가?" 한 문장 정의

2. 리서처가 사전 리서치 브리핑을 공유한다
   → 검색 트렌드, 경쟁 콘텐츠, 타겟 인사이트, 콘텐츠 갭
```

### Phase 2: 자유 발산 (전원 참여)
```
3. 전원이 자유롭게 아이디어를 낸다
   → 비판 금지, 양 우선
   → 각자의 전문 관점에서 아이디어 제시:
     - 크리에이티브 디렉터: 콘셉트·포맷
     - 리서처: 데이터 기반 제안
     - 라이터: 스토리·후킹 아이디어
     - 디자이너: 비주얼·연출 아이디어
     - 커뮤니티: 시청자 니즈 기반 제안
     - 커머스: 수익화 연결 아이디어
```

### Phase 3: 클러스터링 & 선별 (크리에이티브 디렉터 주도)
```
4. 유사 아이디어를 묶고 카테고리화한다
5. 각 카테고리에서 가장 강한 아이디어를 투표/선별한다
   → 선별 기준: 공감도 > 차별성 > 제작 가능성
```

### Phase 4: 구체화 (역할별 심화)
```
6. 라이터: 선별된 콘셉트의 스크립트 구조 설계
7. 디자이너: 썸네일 콘셉트 + 연출 방향 제안
8. 커뮤니티: 시청자 반응 예측 + 참여 유도 방안
9. 커머스: 구매 루트 + 수익화 전략 설계
10. 리서처: 최종 키워드·SEO 추천
```

### Phase 5: 최종 확정 (크리에이티브 디렉터)
```
11. 크리에이티브 디렉터가 모든 의견을 종합하여 최종 콘셉트 확정
12. 각 팀원에게 실행 태스크 배분
```

## 사용 방법

### 기본 사용
주제를 입력하면 6명의 에이전트가 순서대로 브레인스토밍에 참여합니다:

```
[사용자] 주제: "퇴근 후 운동 습관 만들기"

→ 크리에이티브 디렉터: 방향성 설정
→ 리서처: 데이터 브리핑
→ 전원: 자유 발산
→ 라이터: 스크립트 구조화
→ 디자이너: 비주얼 제안
→ 커뮤니티: 시청자 관점 피드백
→ 커머스: 수익화 연결
→ 크리에이티브 디렉터: 최종 정리
```

### 개별 에이전트 활용
특정 관점만 필요한 경우 개별 에이전트를 지정할 수 있습니다:

```
"리서처 관점에서 이 주제 분석해줘"
"커머스 매니저로서 이 영상의 수익화 방안 제안해줘"
"라이터로서 이 콘셉트의 스크립트 구조 잡아줘"
```
